Signal Forge Quick Reference

What You'll Find Here: Commands, options, and workflows to generate content with Signal Forge. Keep this handy while working.

Quick Start

Get Signal Forge running in 3 steps:

  1. Install Signal Forge
git clone <repo>
cd signal-forge
npm install
  1. Configure your API keys
cp .env.example .env

Edit .env and add at least one API key:

ANTHROPIC_API_KEY=your_key_here
OPENAI_API_KEY=your_key_here
GOOGLE_API_KEY=your_key_here
PERPLEXITY_API_KEY=your_key_here
  1. Generate your first document
forge generate deck --input meeting-notes.md --format pptx

Commands

Core Commands

Command Description Example
forge generate <type> Generate content from input forge generate deck -i notes.md
forge agent generate <type> Agentic workflow with research forge agent generate deck -i notes.md --research
forge agent research "<topic>" Research only, no generation forge agent research "AI governance"
forge agent publish <file> Publish to multiple formats forge agent publish deck.md -f pptx,pdf
forge --help Show all commands forge --help

Options Reference

Standard Options

Option Short Values Default Description
--input -i file path required Input markdown file
--output -o file/directory ./output Output location
--provider -p anthropic, openai, google, perplexity anthropic AI provider to use
--format -f pptx, docx, pdf, html, markdown, slides type-specific Output formats (comma-separated)
--mode -m thought-leadership, architecture, advisory auto-detected Content mode
--audience text none Target audience description
--no-edit flag false Skip interactive editor review

Agentic Workflow Options

Option Values Default Description
--research flag false Run research agent before generation
--iterate flag false Enable feedback loop until voice approved
--max-iterations number 5 Maximum revision attempts
--memory flag false Enable long-term memory persistence
--verbose flag false Show detailed progress logs

Output Formats

Specify multiple formats:

forge generate deck -i notes.md -f pptx,html,pdf

Content Modes

Signal Forge auto-detects mode from your input. Override with --mode if needed.

Advisory Mode

Auto-triggers on:

  • Keywords: recommend, strategy, stakeholder, investment, roadmap, executive
  • Audience: executives, leadership, board, clients

Characteristics:

  • Strategic recommendations
  • Executive-friendly language
  • Action-oriented conclusions
  • Stakeholder considerations

Example:

forge generate deck -i strategy-notes.md --mode advisory -f pptx

Architecture Mode

Auto-triggers on:

  • Keywords: system, API, database, component, deployment, infrastructure
  • Audience: engineers, developers, architects, technical teams

Characteristics:

  • Technical depth
  • System diagrams (described in text)
  • Implementation details
  • Trade-off analysis

Example:

forge generate architecture -i design-notes.md --mode architecture -f html,pdf

Thought Leadership Mode

Auto-triggers on:

  • Keywords: I think, exploring, reflecting, questioning, observing
  • Structure: observations, open questions, personal insights

Characteristics:

  • Personal voice
  • Exploratory tone
  • Question-driven
  • Experience-based

Example:

forge generate pov -i observations.md --mode thought-leadership -f html

Voice Score Interpretation

Signal Forge scores generated content for voice authenticity (0-10 scale).

Score Range Status Recommended Action
7.0 - 10.0 ✅ Approved Content ready to use
5.0 - 6.9 ⚠️ Review Needed Consider using --iterate flag
0.0 - 4.9 ❌ Issues Check mode detection, add context to input

Improve low scores:

# Add iteration for automatic improvement
forge agent generate deck -i notes.md --iterate

# Explicitly set mode
forge generate deck -i notes.md --mode advisory

# Add more context to your input file

Common Workflows

1. Meeting Notes to Strategy Deck

# Step 1: Capture meeting notes in markdown
# Step 2: Generate deck
forge generate deck -i meeting-recap.md -f pptx,html

# Step 3: Open output
open output/decks/meeting-recap.pptx

Expected output: 10-15 slide deck with executive summary, key points, recommendations, next steps.

2. Research-Backed Paper

# Step 1: Research topic
forge agent research "AI governance frameworks 2024" -o research.json

# Step 2: Write initial observations in markdown
# Step 3: Generate paper with research
forge agent generate paper -i observations.md --research --iterate -f pdf,html

# Step 4: Review output
open output/papers/observations.pdf

Expected output: 3000-8000 word paper with research citations, structured arguments, conclusions.

3. Technical Design to Architecture Document

# Step 1: Document design session notes
# Step 2: Generate architecture doc
forge generate architecture -i design-session.md --mode architecture -f html,pdf

# Step 3: Review in browser
open output/architecture/design-session.html

Expected output: 15-30 page architecture document with system overview, component descriptions, integration points, trade-offs.

4. Observations to Thought Leadership POV

# Step 1: Write personal observations
# Step 2: Generate POV piece
forge generate pov -i observations.md --mode thought-leadership -f html,markdown

# Step 3: Publish
open output/povs/observations.html

Expected output: 800-1200 word article with personal voice, exploratory questions, insights.

5. Full Agentic Workflow

# Research, generate, iterate until approved
forge agent generate deck -i context.md --research --iterate --verbose -f pptx,html

# Monitor progress in terminal
# Agent will:
# 1. Research topic
# 2. Generate initial draft
# 3. Score voice authenticity
# 4. Iterate if score < 7.0
# 5. Output final approved version

AI Provider Selection

Provider Strengths Best For Speed Cost
Anthropic (Claude) Strategic thinking, voice authenticity Advisory, Thought Leadership Fast $$
OpenAI (GPT-4) General purpose, reliable Technical, mixed content Fast $$
Google (Gemini) Long context windows Papers, architecture docs Medium $
Perplexity Web research, citations Research-heavy content Medium $

Set provider:

forge generate deck -i notes.md --provider anthropic
forge generate architecture -i design.md --provider google

Default: Anthropic (Claude) if API key is configured.

Input Guidelines

What Makes Good Input

Include:

  • Meeting context (who attended, purpose, date)
  • Key decisions and rationale
  • Open questions and concerns
  • Stakeholder perspectives
  • Constraints (budget, timeline, technical)
  • Success criteria

Advisory Mode Input:

# Strategy Meeting - Q1 Planning
Date: 2024-01-15
Attendees: CEO, CTO, VP Product

## Context
Discussing AI integration strategy for 2024...

## Key Points
- Budget: $500K allocated
- Timeline: 6 months
- Stakeholder concern: ROI visibility

## Decisions
1. Start with pilot in customer service
2. Evaluate after 3 months

## Open Questions
- Which vendor to partner with?
- How to measure success?

Architecture Mode Input:

# Payment Service Design
Date: 2024-01-15

## Requirements
- Process 10K transactions/minute
- 99.99% uptime
- PCI DSS compliant

## Constraints
- Existing PostgreSQL database
- Must integrate with Stripe and PayPal
- 3-month timeline

## Non-Functional Requirements
- < 200ms response time
- Horizontal scalability
- Zero-downtime deployments

Thought Leadership Input:

# Observations on AI Adoption

I've been thinking about why some teams adopt AI tools quickly while others resist...

## What I'm Noticing
- Teams with psychological safety experiment more
- Fear of replacement vs. fear of irrelevance
- The "AI skeptic" often becomes the biggest advocate

## Questions I'm Exploring

- What makes someone shift from skeptic to advocate?

## Personal Experience
Last quarter, I watched our most skeptical engineer...

Troubleshooting

Wrong Tone or Style

Problem: Output doesn't match expected voice or style.

Solutions:

# Explicitly set mode
forge generate deck -i notes.md --mode advisory

# Add iteration for refinement
forge agent generate deck -i notes.md --iterate

# Specify audience
forge generate deck -i notes.md --audience "executive leadership team"

Low Voice Score (< 7.0)

Problem: Voice authenticity score is below approval threshold.

Solutions:

# Enable automatic iteration
forge agent generate deck -i notes.md --iterate --max-iterations 10

# Add more context to input file
# Include personal observations, stakeholder quotes, specific details

# Check mode detection
forge generate deck -i notes.md --mode thought-leadership --verbose

No Output Generated

Problem: Command runs but no files appear in output directory.

Solutions:

# Verify API keys
cat .env  # Check keys are present and valid

# Verify input file exists
ls -la input/your-file.md

# Run with verbose flag
forge generate deck -i notes.md --verbose

# Check output directory permissions
ls -la output/

Missing Diagrams in Architecture Docs

Problem: Architecture document describes diagrams but doesn't include visuals.

Explanation: Signal Forge describes diagrams in text. You create the visuals.

Solutions:

# Generate architecture doc
forge generate architecture -i design.md -f html

# Use diagram descriptions to create visuals in:
# - Excalidraw (recommended)
# - Mermaid
# - Draw.io
# - Lucidchart

# Insert visuals into final document

Format Not Supported

Problem: Requested output format fails or is missing.

Solutions:

# Check available formats for content type
forge generate deck --help

# Use comma-separated list
forge generate deck -i notes.md -f pptx,html,pdf

# Verify format spelling
# Correct: pptx, docx, pdf, html, markdown
# Incorrect: powerpoint, word, htm, md

API Rate Limits

Problem: Generation fails with rate limit error.

Solutions:

# Switch to different provider
forge generate deck -i notes.md --provider openai

# Wait and retry (rate limits reset)
sleep 60 && forge generate deck -i notes.md

# Use --verbose to see detailed error
forge generate deck -i notes.md --verbose
