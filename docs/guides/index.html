Signal Forge Documentation

Transform rough notes into polished professional content with AI-powered voice matching.

Quick Start

Get Signal Forge running in 60 seconds:

# 1. Install dependencies
npm install

# 2. Configure API keys
cp .env.example .env
# Edit .env with your OpenAI/Anthropic API key

# 3. Generate your first document
forge generate deck --input meeting-notes.md --format pptx,html

Expected output: A professional slide deck in both PowerPoint and HTML formats, ready to present.

Content Modes Overview

Signal Forge automatically applies the right voice and structure based on your content type.

Mode Use When Voice Characteristics Output Types
Executive Advisory Presenting strategy, making recommendations Confident, consultant-style, action-oriented deck, brief, roadmap
Solution Architecture Writing technical documentation Precise, definitive, implementation-focused architecture, adr, spec
Thought Leadership Sharing ideas and opinions Provisional, questioning, exploratory pov, paper, post
Documentation User guides, tutorials, references Instructional, direct, copy-paste ready guide, reference, tutorial

How It Works

  1. You provide: Raw notes, meeting recaps, observations
  2. Signal Forge analyzes: Content type determines voice mode
  3. You receive: Polished document with appropriate structure and tone
# Same input, different voices
forge generate deck --input notes.md        # Confident recommendations
forge generate architecture --input notes.md # Precise technical detail
forge generate pov --input notes.md         # Questioning exploration

Why Signal Forge?

The problem: Generic AI writers produce generic voice. A blog post tone doesn't work for architecture documents. Technical detail overwhelms executives. Strategy decks need confidence; thought pieces need questions.

The solution: Signal Forge matches voice to content type automatically.

Traditional AI Writing Signal Forge
One generic voice Four specialized modes
Manual formatting Auto-structured outputs
Text-only Multi-format (PPTX, HTML, DOCX, PDF)
Requires prompt engineering Content type = automatic voice

Technical Documentation

Thought Leadership

# Observations to opinion piece
forge generate pov --input observations.md --format md

Research-Enhanced Content

# Topic + research to comprehensive paper
forge agent generate paper --input topic.md --research --format html
