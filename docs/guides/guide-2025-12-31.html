Signal Forge User Guide

Generate professional content with AI-powered voice consistency. Signal Forge transforms your meeting notes, technical discussions, and observations into polished decks, briefs, and documentation—automatically applying the right voice and structure for your content type.

Quick Start

Signal Forge delivers professional documents in 60 seconds.

1. Install Signal Forge

npm install -g signal-forge

2. Configure Your API Key

Create a .env file in your working directory:

ANTHROPIC_API_KEY=your_api_key_here
DEFAULT_AI_PROVIDER=anthropic

3. Generate Your First Document

# Create a simple input file
echo "Meeting with client about cloud migration strategy. 
Key concerns: cost, security, timeline. 
Current state: on-premise datacenter." > input.md

# Generate a strategy deck
forge generate deck --input input.md --format pptx,html

Expected output:

✓ Analyzing input content...
✓ Detecting mode: Executive Advisory
✓ Generating deck content...
✓ Voice check passed (score: 8.5/10)
✓ Exporting to PowerPoint...
✓ Exporting to HTML...

Generated files:
  - output.pptx
  - output.html

Signal Forge has created a professional strategy deck ready for presentation.

Core Concepts

Content Modes

Signal Forge operates in three distinct modes, each with its own voice, structure, and quality standards:

Mode Use For Voice Style Output Types
Thought Leadership Blog posts, POVs, articles Narrative, exploratory, question-led pov, paper, post
Solution Architecture Technical docs, system designs Precise, definitive, diagram-heavy architecture, adr, tech-deck, spec
Executive Advisory Strategy decks, briefs Confident consultant, outcome-focused deck, brief, roadmap

Signal Forge auto-detects the appropriate mode from your input, or you can specify it explicitly with the --mode flag.

Voice Consistency

Every document passes through a voice checker that scores content on a 0-10 scale:

  • 7-10: Production ready, voice principles applied correctly
  • 4-6: Needs revision, some voice inconsistencies
  • 0-3: Major issues, requires significant rework

The agentic workflow automatically iterates until reaching a score of 7 or higher.

Agentic Workflow

Standard generation creates content in one pass. Agentic workflow adds:

  1. Research phase: Gathers context before writing
  2. Iteration loop: Refines content based on voice checker feedback
  3. Memory: Learns from previous documents to improve consistency

Use the generate command to create content in a single pass:

forge generate <type> --input <file> [options]

Available content types:

Type Mode Description

post Thought Leadership Blog post
architecture Solution Architecture System architecture document

Standard Options

forge generate deck \
  --input meeting-notes.md \
  --output strategy-deck \
  --format pptx,html,pdf \
  --provider anthropic \
  --audience "C-suite executives"

--format -f Export formats (comma-separated) html

--audience Target audience description General
--no-edit Skip interactive editor review false

Enable research and iterative refinement for higher-quality output.

Basic Agentic Generation

Agentic Options

Option Description Default
--research Run research agent before generation Disabled
--iterate Enable feedback loop until voice approval Disabled
--max-iterations Maximum revision attempts 5
--memory Enable cross-document learning Disabled

Working with Content Modes

Thought Leadership Mode

Create exploratory, narrative content that shows your thinking process.

When to use:

  • Blog posts exploring emerging trends
  • Conference presentations with new ideas
  • POV documents on industry shifts
  • Papers that challenge conventional thinking

Voice principles applied:

  • Opens with tension or questions, never thesis statements
  • Shows evolution of thinking, not just conclusions
  • Includes self-interrogation without self-doubt
  • Ends provisionally: "Here's what I think today"
  • Uses intentional fragments for rhythm
  • Grounds in actual experience, not theory

Example input (observations.md):

I've been noticing something odd in client conversations lately. 
Everyone's talking about AI governance, but nobody's actually doing it.

Three conversations this week:
- Retail CIO: "We need guardrails" (has no guardrails)
- Manufacturing VP: "Compliance is critical" (no compliance framework)
- Healthcare CISO: "We're being thoughtful" (no documented policies)

What's going on here? Is this just early-stage chaos, or something deeper?

Generate thought leadership:

forge generate pov \
  --input observations.md \
  --mode thought-leadership \
  --format docx,html

Expected output structure:

Title: The AI Governance Paradox

Opening (tension):
"Everyone's talking about AI governance. Nobody's doing it. 
Why the gap between rhetoric and reality?"

Body (show the work):
- The three conversations that sparked this
- What I initially thought (and why I was wrong)
- The pattern I'm seeing now
- Why this matters more than it seems

Closing (provisional):
"Here's where I've landed for now: [insight]
But I'm watching for..."

Solution Architecture Mode

Create precise, definitive technical documentation.

When to use:

  • System architecture documents
  • Architecture Decision Records (ADRs)
  • Technical specifications
  • API documentation
  • Infrastructure designs

Voice principles applied:

  • Leads with conclusions, not questions
  • Definitive statements: "The system uses X" not "Here's where I've landed"
  • Complete sentences, no fragments
  • Diagrams over prose
  • Reference-grade precision (implementable from docs)
  • No provisional language

Example input (system-design.md):

Need to document the new microservices architecture for the platform team.

Key decisions:
- Event-driven communication between services
- PostgreSQL for transactional data
- Redis for caching
- Kubernetes for orchestration
- API Gateway pattern for external access

Non-functional requirements:
- 99.9% uptime
- Sub-200ms response time
- Handle 10K requests/second

Generate architecture documentation:

forge generate architecture \
  --input system-design.md \
  --mode architecture \
  --format html,pdf

Expected output structure:

Title: Platform Microservices Architecture

Overview:
The system implements an event-driven microservices architecture 
using Kubernetes orchestration.

Architecture Diagram:
[ASCII or described diagram of system components]

Components:
- API Gateway: Kong (handles routing, auth, rate limiting)
- Services: 12 microservices (detailed below)
- Data Layer: PostgreSQL + Redis
- Message Bus: Apache Kafka

Service Specifications:
[Detailed specs for each service]

Non-Functional Requirements:
- Availability: 99.9% (measured monthly)
- Performance: <200ms p95 response time
- Scalability: 10K req/sec sustained load

Executive Advisory Mode

Create confident, outcome-focused strategic recommendations.

When to use:

  • Strategy decks for client presentations
  • Executive briefs on transformation opportunities
  • Digital roadmaps
  • Business case documents
  • Board-level recommendations

Voice principles applied:

  • Leads with business outcomes, not technical details
  • Confident recommendations: "I recommend..." not "You might consider..."
  • Pattern recognition: "I've seen this across retail, manufacturing..."
  • Grounds in client context: References actual conversations
  • Structured clarity: Scannable headers, tables, bullets
  • Consultant perspective: External advisor, not internal team

Example input (client-meeting.md):

Meeting with RetailCo CIO and VP Engineering
Date: Jan 15, 2024

Current state:
- Monolithic e-commerce platform (8 years old)
- Scaling issues during peak season
- 6-month release cycles
- Cloud migration stalled for 2 years

Key concerns raised:
- "Can't keep up with competitors' feature velocity"
- "Black Friday crashes cost us $2M last year"
- "Engineering team frustrated with slow deployments"

Budget: $5M for transformation
Timeline: Must show results by Q3

Generate strategy deck:

forge generate deck \
  --input client-meeting.md \
  --mode advisory \
  --format pptx,html \
  --audience "CIO and VP Engineering"

Expected output structure:

Slide 1: Executive Summary
"I recommend a phased cloud-native transformation 
targeting $2M+ in avoided downtime costs by Q3"

Slide 2: Current State Assessment
[Table showing pain points and business impact]

Slide 3: Recommended Approach
Three-phase transformation:
- Phase 1 (Q1): Stabilize and lift-shift critical services
- Phase 2 (Q2): Modernize checkout and inventory
- Phase 3 (Q3): Complete migration and optimization

Slide 4: Why This Approach
"I've seen this pattern work across 12 retail transformations.
The phased approach delivers early wins while managing risk."

Slide 5: Expected Outcomes
[Table with metrics, timeline, investment]

Slide 6: Risks and Mitigations
[Structured risk assessment]

Slide 7: Next Steps
"Start with Phase 1 kickoff next week..."

Mode Override

Force a specific mode when auto-detection is not appropriate:

# Force thought leadership voice for technical content
forge generate tech-deck \
  --input technical-notes.md \
  --mode thought-leadership \
  --format pptx

# Force advisory voice for internal documentation
forge generate paper \
  --input research.md \
  --mode advisory \
  --format docx

Input Best Practices

What Makes Good Input

Signal Forge works best with context-rich input that includes:

  1. Situation: What's happening, who's involved
  2. Content: Key points, decisions, observations
  3. Constraints: Timeline, budget, audience
  4. References: Links, prior documents, examples

Input Quality Checklist

Before generating, verify your input includes:

  • Clear context (who, what, when, why)
  • Specific details (not generic statements)
  • Stakeholder perspectives (quotes, concerns)
  • Constraints (budget, timeline, technical)
  • References (links, prior docs, examples)
  • Desired outcomes (what success looks like)

Poor input example:

Need a deck about cloud migration.
Client wants to move to AWS.
Should be done soon.
