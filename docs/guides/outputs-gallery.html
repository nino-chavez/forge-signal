Signal Forge Output Gallery

Preview what Signal Forge generates for each content type and mode. Understand structure, voice, and quality indicators.

Understanding Your Output

Run Signal Forge with any content type to see its output:

forge generate deck --input "modernize legacy platform"

Check the voice score in the console output:

✓ Content generated successfully
  Voice Score: 8.2/10 (Executive Advisory)
  Output: output/strategic-modernization-deck.pptx

Voice scores above 7.0 are ready to use. Scores below 5.0 indicate the wrong mode was detected.

Strategic Deck

Generate a strategy presentation:

forge generate deck \
  --input "cloud migration strategy for retail company" \
  --format pptx

Expected output structure:

Slide 1:  Cover (title, date, confidential marking)
Slide 2:  Executive Summary
Slide 3:  Current State Assessment
Slide 4:  Key Challenges
Slide 5:  Strategic Approach
Slide 6:  Phase 1 Details
Slide 7:  Phase 2 Details
Slide 8:  Investment & Timeline
Slide 9:  Risk Mitigation
Slide 10: Next Steps

Voice characteristics:

  • Confident recommendations: "I recommend a three-phase approach..."
  • Pattern recognition: "We've seen this work across similar transformations..."
  • Business outcomes: "This delivers 40% cost reduction..."
  • Scannable bullets: Short, declarative statements

Sample slide content:

RECOMMENDATION: PHASED CLOUD MIGRATION

I recommend a three-phase approach based on
successful patterns from retail transformations:

Phase 1: Foundation (Q1-Q2)
- Establish AWS infrastructure
- Migrate dev/test environments
- Build cloud competency

Phase 2: Core Systems (Q3-Q4)
- E-commerce platform migration
- Customer data consolidation
- API gateway implementation

This approach minimizes disruption while
delivering early wins.

Executive Brief

Generate a 1-3 page executive summary:

forge generate brief \
  --input "digital transformation initiative approval" \
  --format docx

Expected structure:

1. Headline Summary (2-3 sentences)
2. Situation Overview (1 paragraph)
3. Key Findings (3-5 bullets)
4. Recommendations (numbered list)
5. Investment & Timeline (table)
6. Decision Request (clear ask)

Voice characteristics:

  • Lead with conclusion: "I recommend proceeding with $4M Phase 1 funding..."
  • Dense, information-rich: Every sentence delivers value
  • No preamble: Jumps straight to the point
  • Clear ask: "I need approval for Phase 1 by January 15"

Strategic Roadmap

Generate an initiative roadmap:

forge generate roadmap \
  --input "platform modernization over 18 months" \
  --format html

Expected structure:

1. Executive Summary
2. Strategic Objectives
3. Phased Approach Overview
4. Phase 1 Breakdown (timeline, deliverables, resources)
5. Phase 2 Breakdown
6. Phase 3 Breakdown
7. Dependencies & Critical Path
8. Resource Requirements
9. Success Metrics
10. Governance Model

Voice characteristics:

  • Definitive timelines: "Phase 1 completes Q2 2025"
  • Clear ownership: "Platform team leads, Security reviews"
  • Measurable milestones: "Deploy to production with <100ms latency"

Architecture Document

Generate comprehensive technical documentation:

forge generate architecture \
  --input "microservices e-commerce platform with event streaming" \
  --format html

Expected structure (arc42 standard):

1.  Introduction and Goals
2.  Constraints
3.  Context and Scope
4.  Solution Strategy
5.  Building Block View
    - System Context
    - Container View
    - Component View
6.  Runtime View
7.  Deployment View
8.  Crosscutting Concepts
9.  Architecture Decisions
10. Quality Requirements
11. Risks and Technical Debt

Voice characteristics:

  • Definitive statements: "The system uses Kafka for event streaming"
  • No hedging: Never "might" or "could potentially"
  • Implementation-ready: Specific enough to build from
  • Quantified requirements: "99.9% availability, <200ms p95"

Architecture Decision Record

Generate a structured decision record:

forge generate adr \
  --input "choosing Kafka over RabbitMQ for event streaming" \
  --format html

Expected structure:

ADR-001: Event Streaming Platform Selection

Status: Accepted
Date: 2024-11-25

Context:
  The new architecture requires event-driven communication
  between microservices with strict ordering guarantees...

Decision:
  We will use Apache Kafka as the event streaming platform.

Consequences:
  Positive:
  - High throughput (100K+ msgs/sec)
  - Event replay capability
  - Strong ordering per partition

  Negative:
  - Operational complexity
  - Requires Zookeeper (or KRaft in newer versions)

Technical Specification

Generate an implementation specification:

forge generate spec \

  --format html

Expected structure:

  • Overview and scope
  • Functional requirements
  • API endpoints (with request/response examples)
  • Data models
  • Security considerations
  • Error handling
  • Testing requirements

Point of View (POV)

Generate a thought leadership piece:

forge generate pov \
  --input "observations on AI governance" \
  --format html,markdown

Expected structure:

800-1200 words, flowing prose:

Opening: Observation that creates tension
  "Everyone's building AI governance frameworks.
   Nobody's using them."

Middle: Exploration of theories
  - Theory 1: Frameworks too rigid
  - Theory 2: Wrong incentives
  - Self-interrogation: "But maybe I'm wrong..."

Ending: Provisional conclusion
  "Here's where I've landed for now..."

Voice characteristics:

  • Opens with observation, not thesis
  • Shows thinking evolution
  • Includes self-interrogation
  • Ends provisionally
  • Uses intentional fragments
  • Avoids corporate jargon

Long-Form Paper

Generate an in-depth paper:

forge agent generate paper \

  --format pdf

  • 3000-8000 words
  • Research citations integrated
  • Structured sections with clear progression
  • Thought leadership voice throughout

Voice Quality Scores

Signal Forge scores generated content for voice authenticity (0-10 scale).

Score Range Status Recommended Action
7.0 - 10.0 Approved Content ready to use
5.0 - 6.9 Review Needed Consider using --iterate flag
0.0 - 4.9 Issues Check mode detection, add context

What Affects Voice Score

  • Opening style: Matches mode expectations
  • Tone consistency: Maintains voice throughout
  • Language patterns: Mode-appropriate phrasing
  • Structure alignment: Follows expected format
  • Audience fit: Appropriate for target readers
