Signal Forge User Guide

Overview

Signal Forge transforms your meeting notes, observations, and raw ideas into polished professional content. Choose from three content modes—thought leadership, solution architecture, or executive advisory—and Signal Forge applies the appropriate voice, structure, and quality standards automatically. Export to PowerPoint, Word, PDF, HTML, or Google Slides.

Quick Start

Get your first document generated in 60 seconds:

# 1. Install Signal Forge
npm install -g signal-forge

# 2. Configure your API key
echo "ANTHROPIC_API_KEY=your_key_here" > .env

# 3. Create an input file
echo "Meeting recap: Discussed AI governance challenges..." > input.md

# 4. Generate a strategy deck
forge generate deck --input input.md --format pptx

# Output: strategy-deck.pptx created

Your deck is ready. Open the .pptx file to review.

Core Concepts

Content Modes

Signal Forge operates in three distinct modes. Each mode applies different voice principles, structures, and quality standards:

Mode Purpose Voice Style Output Types
Thought Leadership Blog posts, articles, POVs Narrative, provisional, question-led pov, paper, post
Solution Architecture Technical docs, system designs Precise, definitive, reference-grade architecture, adr, tech-deck, spec
Executive Advisory Strategy decks, executive briefs Confident consultant, outcome-focused deck, brief, roadmap

Signal Forge auto-detects the appropriate mode from your content type. Override with the --mode flag when needed.

Voice Principles by Mode

Thought Leadership:

  • Opens with tension or questions, never thesis statements
  • Shows evolution of thinking, not just conclusions
  • Ends provisionally: "Here's what I think today"
  • Uses intentional fragments for rhythm
  • Grounds in actual experience

Solution Architecture:

  • Leads with conclusions, not questions
  • Uses definitive language: "The system uses X"
  • Employs complete sentences, no fragments
  • Prioritizes diagrams over prose
  • Maintains reference-grade precision

Executive Advisory:

  • Leads with business outcomes
  • Makes confident recommendations: "I recommend..."
  • Recognizes patterns: "I've seen this across retail, manufacturing..."
  • References actual client context
  • Uses scannable structure: headers, tables, bullets

Quality Gates

Signal Forge includes a voice checker that scores content on a 0-10 scale:

  • 0-4: Major voice issues, requires significant revision
  • 5-6: Minor issues, targeted fixes needed
  • 7-8: Good quality, ready for review
  • 9-10: Excellent, publish-ready

The agentic workflow iterates automatically until reaching a score of 7 or higher.

Basic Usage

Generate Command Structure

forge generate <type> [options]

Standard Options

Option Description Example
-i, --input <file> Input file path --input notes.md
-o, --output <file> Output file path --output strategy.pptx
-p, --provider <name> AI provider --provider anthropic
-f, --format <formats> Output formats (comma-separated) --format pptx,pdf,html
-m, --mode <mode> Content mode --mode advisory
--audience <audience> Target audience --audience "C-suite executives"
--no-edit Skip editor review --no-edit

Agentic Workflows

Agentic Generate Command

The agentic workflow adds research and iterative refinement:

forge agent generate <type> [options]

Agentic Options

Option Description Default
--research Enable research agent before generation Off
--iterate Enable feedback loop until voice approval Off
--max-iterations <n> Maximum revision iterations 5
--memory Enable long-term memory persistence Off

forge generate deck \

Best Practices

Input Quality

Include relevant context:

# Good Input Example

## Meeting Context
- Client: Acme Corp (Fortune 500 retail)
- Attendees: CTO Sarah Chen, VP Engineering Mike Rodriguez
- Date: 2024-01-15
- Purpose: Quarterly architecture review

## Key Discussion Points
[Your notes here]

## Decisions Made
[Decisions with rationale]

## Open Questions
[Unresolved items]

What to include:

  • Who was involved and their roles
  • What decisions were made and why
  • Any constraints or requirements
  • Stakeholder concerns and questions
  • Reference materials or prior context

What to avoid:

  • Vague or incomplete notes
  • Missing context about audience
  • Unclear objectives
  • No indication of desired outcome

Mode Selection

Let auto-detection work:

# Signal Forge detects mode from content type
forge generate deck --input notes.md

Override when needed:

# Force architecture mode for technical deck
forge generate deck --input notes.md --mode architecture

When to override:

  • Hybrid documents spanning multiple modes
  • Specific voice requirements
  • Auto-detection produces wrong tone

Iteration Strategy

Start with basic generation:

forge generate deck --input notes.md --format pptx

Add iteration if voice is off:

forge agent generate deck \
  --input notes.md \
  --iterate \
  --max-iterations 3 \
  --format pptx

Add research for depth:

forge agent generate paper \
  --input observations.md \
  --research \
  --iterate \
  --format pdf

Output Review

Review checklist:

  1. Voice alignment: Does the tone match the mode?
  2. Content accuracy: Are facts and context correct?
  3. Audience fit: Is the technical depth appropriate?
  4. Structure: Is the document scannable and logical?
  5. Completeness: Are all key points addressed?

Iteration workflow:

# 1. Generate initial version
forge generate deck --input notes.md --format pptx

# 2. Review output
# 3. Add more context to input file if needed
# 4. Regenerate with iteration

forge agent generate deck \
  --input notes-updated.md \
  --iterate \
  --format pptx

Troubleshooting

No Output Generated

Symptoms:

  • Command completes but no files created
  • Error message: "Generation failed"

Solutions:

  1. Check API keys:
# Verify .env file exists
cat .env

# Test API key
forge test-connection --provider anthropic
  1. Verify input file:
# Check file exists and is readable
cat input.md

# Check file permissions
ls -la input.md
  1. Check API credits:
# Test with minimal input
echo "Test content" > test.md
forge generate pov --input test.md --format md
  1. Enable debug logging:
forge generate deck --input notes.md --debug

Wrong Voice or Tone

Symptoms:

  • Output sounds too formal/informal
  • Wrong opening style (question vs. conclusion)
  • Inconsistent tone throughout

Solutions:

  1. Use explicit mode flag:
# Force thought leadership mode
forge generate deck --input notes.md --mode thought-leadership

# Force architecture mode
forge generate deck --input notes.md --mode architecture
  1. Add audience context:
forge generate deck \
  --input notes.md \
  --audience "C-suite executives, non-technical"
  1. Enable iteration:
forge agent generate deck \
  --input notes.md \
  --iterate \
  --max-iterations 5
  1. Add more context to input:
# Add to top of input file

## Intended Audience
C-suite executives, non-technical background

## Desired Tone
Confident consultant, outcome-focused, minimal jargon

## Key Message
We need to act now on AI governance

Format Export Failures

Symptoms:

  • Some formats generate, others fail
  • Error: "Export to PPTX failed"

Solutions:

  1. Check format-specific dependencies:
# PowerPoint requires python-pptx
pip install python-pptx

# PDF requires wkhtmltopdf
brew install wkhtmltopdf  # macOS
apt-get install wkhtmltopdf  # Linux
  1. Test formats individually:
# Test each format separately
forge generate deck --input notes.md --format pptx
forge generate deck --input notes.md --format pdf
forge generate deck --input notes.md --format html
  1. Use publish command:
# Generate markdown first, then publish
forge generate deck --input notes.md --format md
