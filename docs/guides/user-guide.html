Signal Forge User Guide

Overview

Signal Forge transforms your raw notes, observations, and meeting recaps into polished professional content. It understands three distinct content modes—thought leadership, solution architecture, and executive advisory—and applies the right voice, structure, and workflow automatically. Generate strategy decks, technical documentation, and thought leadership pieces that maintain consistent quality and appropriate tone.

Quick Start

Get your first document generated in 60 seconds:

# 1. Install Signal Forge
npm install -g signal-forge

# 2. Configure your API key
echo "ANTHROPIC_API_KEY=your_key_here" > .env

# 3. Create a simple input file
echo "Meeting recap: Discussed AI governance strategy for Q2..." > notes.md

# 4. Generate your first deck
forge generate deck --input notes.md --format pptx

Expected output:

✓ Analyzing input content...
✓ Detected mode: Executive Advisory
✓ Generating deck structure...
✓ Creating slides...
✓ Voice check passed (score: 8.2/10)
✓ Generated: output/deck-2024-01-15.pptx

Installation and Setup

1. Install Signal Forge

npm install -g signal-forge

2. Configure API Keys

Create a .env file in your working directory:

# Choose at least one provider
ANTHROPIC_API_KEY=sk-ant-xxxxx
OPENAI_API_KEY=sk-xxxxx
GOOGLE_API_KEY=xxxxx

# Optional: For Google Slides export
GOOGLE_SLIDES_CREDENTIALS_PATH=./credentials.json

3. Verify Installation

forge --version
forge --help

Expected output:

Usage: forge [command] [options]

Core Concepts

Content Modes

Signal Forge operates in three distinct modes, each with specific voice principles and output requirements:

Mode Purpose Voice Style Output Types
Thought Leadership Explore ideas, share insights Narrative, provisional, question-led POVs, blog posts, papers
Solution Architecture Document technical systems Precise, definitive, reference-grade Architecture docs, ADRs, specs
Executive Advisory Strategy and recommendations Confident consultant, outcome-focused Decks, briefs, roadmaps

Mode Detection

Signal Forge automatically detects the appropriate mode from your input content. Override when needed:

# Auto-detect mode (recommended)
forge generate deck --input notes.md

# Force specific mode
forge generate deck --input notes.md --mode advisory

Voice Quality

Every document receives a voice quality score (0-10 scale). The system checks for:

  • Appropriate opening style for the mode
  • Consistent tone throughout
  • Mode-appropriate language patterns
  • Structural alignment
  • Audience fit

Documents scoring below 7 trigger automatic revision in agentic workflows.

Basic Generation

Generate Command Structure

forge generate <type> [options]

Available Document Types

Type Mode Description
pov Thought Leadership Point of view article or blog post
post Thought Leadership Short-form blog post
paper Thought Leadership Long-form exploration paper
architecture Solution Architecture System architecture documentation
adr Solution Architecture Architecture Decision Record
tech-deck Solution Architecture Technical presentation
spec Solution Architecture Technical specification
deck Executive Advisory Strategy presentation deck
brief Executive Advisory Executive summary brief
roadmap Executive Advisory Strategic roadmap

Standard Options

-i, --input <file>          Input file path (required)
-o, --output <file>         Output file path (optional)
-p, --provider <name>       AI provider: openai, anthropic, google, perplexity
-f, --format <formats>      Output formats (comma-separated)
-m, --mode <mode>           Override mode detection
--audience <description>    Target audience description
--no-edit                   Skip interactive editor review

Agentic Workflows

Agentic Generation

The agentic workflow adds research capabilities and iterative refinement:

forge agent generate <type> [options]

Agentic Options

--research                  Enable research agent before generation
--iterate                   Enable feedback loop until voice approval
--max-iterations <n>        Maximum revision attempts (default: 5)
--memory                    Enable long-term memory persistence

Research-Enhanced Generation

1. Generate with automatic research:

forge agent generate paper \
  --input initial-thoughts.md \
  --research \
  --iterate \
  --format pdf

Expected workflow:

✓ Analyzing input...
✓ Research agent activated
  → Searching: "AI governance frameworks"
  → Searching: "enterprise AI policies"
  → Found 12 relevant sources
✓ Generating draft...
✓ Voice check: 6.8/10 - iterating...
✓ Revision 1: 7.9/10 - approved
✓ Generated: output/paper-2024-01-15.pdf

2. Research-only mode:

forge agent research "AI governance in financial services" \
  --output research-context.json

Expected output:

{
  "query": "AI governance in financial services",
  "sources": [
    {
      "title": "Federal Reserve AI Guidelines 2024",
      "url": "https://...",
      "summary": "...",
      "relevance": 0.94
    }

  "synthesis": "Key themes across sources..."
}

3. Use research context in generation:

forge agent generate brief \
  --input notes.md \
  --research-context research-context.json \
  --format docx

Iteration Workflow

When --iterate is enabled, Signal Forge refines content until voice quality reaches 7+:

forge agent generate deck \
  --input notes.md \
  --iterate \
  --max-iterations 3 \
  --format pptx

Expected iteration flow:

✓ Draft generated
✓ Voice check: 5.2/10
  Issues: Opening too tentative, lacks business context
✓ Revision 1: 6.8/10
  Issues: Recommendations need more confidence
✓ Revision 2: 8.1/10 - approved
✓ Generated: output/deck-2024-01-15.pptx

Output Formats

Format Options

Specify output formats with the --format flag (comma-separated):

forge generate deck --input notes.md --format pptx,html,pdf

Available Formats

Format Extension Best For Notes
PowerPoint .pptx Strategy decks, presentations Editable slides with speaker notes
Word .docx POVs, briefs, papers Full formatting, comments, track changes
PDF .pdf Final distribution Print-ready, locked formatting
HTML .html Web pages, interactive decks Responsive, shareable via URL
Google Slides URL Collaborative presentations Requires Google credentials
Markdown .md Version control, documentation Plain text, Git-friendly

Format-Specific Features

PowerPoint (.pptx):

  • Automatic slide layouts based on content
  • Speaker notes included
  • Charts and tables formatted
  • Theme applied based on mode

Word (.docx):

  • Heading styles for navigation
  • Table of contents auto-generated
  • Comments for review points
  • Track changes enabled

HTML (.html):

  • Responsive design
  • Interactive navigation
  • Embedded diagrams
  • Print stylesheet included

Google Slides:

  --share team@company.com

Mode-Specific Workflows

Thought Leadership Workflow

1. Capture observations:

# observations.md

Been noticing a pattern in client conversations...
Everyone's talking about AI governance, but nobody's actually doing it.

Why? Three things I keep hearing:
- "We don't know where to start"
- "Legal says no to everything"
- "We tried a framework but nobody uses it"

Maybe the problem isn't the frameworks themselves...

2. Generate POV:

forge agent generate pov \
  --input observations.md \
  --research \
  --iterate \
  --format docx,html

3. Review voice characteristics:

  • Opens with observation, not thesis
  • Shows thinking evolution
  • Includes self-interrogation
  • Ends provisionally
  • Uses intentional fragments

Solution Architecture Workflow

1. Document requirements:

# requirements.md

System: Event-driven microservices platform

Functional requirements:
- Process 100K events/second
- Support 20+ event types
- Enable real-time analytics
- Provide event replay capability

Non-functional requirements:
- 99.99% availability
- <100ms latency p95
- Multi-region deployment
- GDPR compliant

2. Generate architecture:

forge generate architecture \
  --input requirements.md \
  --format html,pdf \
  --provider openai

3. Review technical precision:

  • Definitive statements (no "might" or "could")
  • Specific technology choices
  • Quantified requirements
  • Implementable from documentation
  • Diagrams described in detail

Advanced Features

Multi-Format Publishing

Generate multiple formats in one command:

forge agent publish content.md \
  --format pptx,docx,pdf,html,google-slides \
  --output-dir ./deliverables

Expected output:

✓ Generating PowerPoint...
✓ Generating Word document...
✓ Generating PDF...
✓ Generating HTML...
✓ Publishing to Google Slides...
✓ All formats generated in ./deliverables/

Memory Persistence

Enable long-term memory for consistent voice across projects:

forge agent generate deck \
  --input notes.md \
  --memory \
  --project "client-acme"

Memory stores:

  • Voice preferences and patterns
  • Client-specific context
  • Previous document decisions
  • Terminology preferences

View memory:

forge memory show --project "client-acme"

Clear memory:

forge memory clear --project "client-acme"

Custom Voice Profiles

Create reusable voice profiles:

forge voice create "technical-blog" \
  --mode thought-leadership \
  --tone "accessible but precise" \
  --audience "senior engineers" \
  --examples ./voice-samples/

Use custom profile:

forge generate post \
  --input draft.md \
  --voice-profile "technical-blog"

Provider Selection

Provider Comparison

Provider Best For Strengths Limitations
Anthropic (Claude) Strategic thinking, voice consistency Nuanced tone, long context, iterative refinement Higher cost per token
OpenAI (GPT-4) General purpose, technical content Fast, reliable, good at structure Less nuanced voice
Google (Gemini) Long documents, research synthesis Large context window, multimodal Newer, less tested
Perplexity Research-heavy content Web search integration, citations Limited creative voice

Quality Control

Voice Scoring

Every document receives a voice quality score:

forge voice-check document.md --mode advisory

Expected output:

Voice Quality Score: 7.8/10

✓ Opening: Strong (leads with business outcome)
✓ Tone: Consistent (confident consultant throughout)
✓ Language: Appropriate (no provisional hedging)
⚠ Structure: Good (could use more tables for scanning)
✓ Audience: Well-matched (executive-appropriate)

Recommendations:
- Convert bullet lists on slides 3-4 to comparison tables
- Add more pattern recognition references

Manual Voice Review

Review generated content before finalizing:

forge generate deck \
  --input notes.md \
  --format pptx  # Opens in editor for review

Skip review (use with caution):

forge generate deck \
  --input notes.md \
  --format pptx \
  --no-edit

Iteration Strategies

Targeted fixes (for scores 6-7):

forge agent generate deck \
  --input notes.md \
  --iterate \
  --strategy targeted  # Fix specific issues only

Strategic revision (for scores 4-6):

forge agent generate deck \
  --input notes.md \
  --iterate \
  --strategy strategic  # Larger structural changes

Full rewrite (for scores <4):

forge agent generate deck \
  --input notes.md \
  --iterate \
  --strategy rewrite  # Complete regeneration
