# AI-Augmented Product Operations - Quick Start

**For:** Tech Arch stepping into Product + AI Ops role
**Context:** You pioneer builder/operator behavior, work across org boundaries, define new ways of working
**Reference Implementation:** WooCommerce Migration (use as example, not as sole mandate)

---

## Your Role in 3 Sentences

1. You're **not** a traditional PM. You're Tech Arch + Product + AI Ops.
2. You **own outcomes** (product works, customers happy, revenue delivered), not process.
3. You **work in the gaps** where traditional roles don't fit (cross-org, builder/operator hybrid, AI-first).

---

## Decision Rights (Quick Reference)

**YOU OWN:**
- Product vision & roadmap for your domain
- Architecture within your product boundary
- Resource prioritization (when resources allocated to you)
- Go/No-Go for your milestones

**YOU FACILITATE:**
- Cross-product APIs (you propose, Engineering approves)
- Resource requests (you ask, their manager approves)
- Platform standards (you comply, Platform Arch owns)

**YOU OPERATE IN GAPS:**
- BigCommerce + Makeswift can't agree → You broker
- "Not feasible" but customers need it → You find the path
- No one owns the integration → You own it

## The Operating Model (5 Steps)

### 1. VALIDATE HYPOTHESIS (2 hours with AI)
```
AI: Market research, technical feasibility, unit economics
You: Make GO/NO-GO decision
Output: 1-page brief with recommendation
```

### 2. CREATE EPIC (4 hours with AI)
```
AI: Generate user stories, technical tasks, estimates
You: Refine, get resource commitments
Output: Sprint-ready backlog
```

### 3. EXECUTE SPRINT (2 weeks)
```
Daily: Standup, unblock team
AI: Code review, test generation, documentation
You: Decisions, stakeholder mgmt, demos
Output: Working software shipped
```

### 4. DEPLOY TO PRODUCTION (1 day)
```
AI: Pre-flight checklist, monitoring
You: Execute deployment, validate, rollback if needed
Output: Feature live, metrics tracking
```

### 5. MEASURE & LEARN (ongoing)
```
Track: Outcomes (revenue, NPS), execution (cycle time, AI augmentation)
Update: Decision logs at 30/60/90 days
Output: Learnings that improve next cycle
```

## Cross-Org Execution (No Direct Reports)

**When you need 0.5 FTE from another team:**

1. **Validate the need** (AI: "Is this the right ask? Alternatives?")
2. **Make it win-win** ("Here's what's in it for you: [BENEFIT]")
3. **Escalate if blocked** (Manager → Their Manager, verbal, 5 min)
4. **Execute** (Weekly sync, clear commitments, unblock them)

**Never:**
- Manage their performance (that's their manager's job)
- Change priorities mid-sprint
- Bypass their manager

## Common Mistakes

**MISTAKE:** "Let me document the process first"
**FIX:** Execute on real work, document learnings after

**MISTAKE:** "I need everyone's approval before starting"
**FIX:** Bias to action. Prototype, demonstrate, iterate.

**MISTAKE:** "AI will do all the work"
**FIX:** AI provides speed/data. You provide judgment/decisions.

**MISTAKE:** "I can't do this without direct reports"
**FIX:** Cross-org execution via trade-based resource requests

**MISTAKE:** "This is too different from how we work today"
**FIX:** Prove it with one initiative (WooCommerce), then scale
