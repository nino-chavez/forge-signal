1. Context & Problem Statement

Commerce.com operates under public company constraints with 4-tier data classification and IP protection as the primary security concern. Current AI policies address request-response tooling but leave gaps for agentic workflows, multi-model orchestration, and scaled adoption across product teams.

Current State

Dimension Status Gap
Data Classification 4-tier (Public, Internal, Confidential, Restricted) Doesn't address agent context persistence
Tool Approval Selective (some tools approved with restrictions) Per-tool, not per-workflow or orchestration chain
Policy Documentation Exists in Confluence (prompt libraries, custom GPTs) Written for chat paradigm, not agentic operations
Team Maturity Unknown / Unassessed No baseline to plan interventions
The Agentic Gap

Existing policies assume: Human → Prompt → AI → Response → Human reviews.

Agentic reality: Human → Goal → Agent → [Tool → API → File → Tool → ...] → Output.

The policy gaps: autonomous decision chains, multi-tool orchestration, persistent context, and the difference between input exposure and output derivation.

Strategic Objectives

  • Establish governance that enables productivity without creating IP/security risk
  • Create a maturity path from individual experimentation to scaled agentic operations
  • Build quality gates that distinguish draft, review-ready, and shippable artifacts
  • Enable the Product Architect role to guide without owning implementation

4. Maturity Model

A 6-level maturity model for AI operations, aligned with typical enterprise adoption curves. The hypothesis: Commerce.com is at Level 2 organizationally, with pockets at Level 1 and Level 3.

0
Blocked
No AI usage. Active resistance or prohibition.
1
Experimental
Ad-hoc individual usage. No shared patterns.
2
Sanctioned
Approved tools exist. Usage policies defined. Still individual.
3
Practiced
Shared workflows. Prompt libraries. Quality patterns emerge.
4
Integrated
AI in the workflow, not bolted on. Clear quality gates.
5
Agentic
Autonomous agents with oversight. Production guardrails.

7. Workflow Patterns

The Air Gap Pattern

For confidential inputs that cannot directly enter AI context, use a human abstraction layer to sanitize inputs and rehydrate outputs.

Confidential Context (Confluence, internal docs, roadmaps) | v [HUMAN: Abstraction Layer] ← Extract patterns, anonymize, remove specifics | v Safe Input (patterns, structures, public references) | v [AI Agent] ← Operates only on sanitized inputs | v Draft Output (structure, prose, frameworks) | v [HUMAN: Rehydration Layer] ← Add specifics, names, numbers back | v Final Output (classified, ready for intended audience)

Workflow Classification

Tier Data Access Autonomy Checkpoints Example Use Cases
A: Sandbox Public only Full End-of-session Learning, experimentation, public docs
B: Guided Public + Internal Within session Before external action Internal docs, code with abstracted context
C: Supervised Up to Confidential (via air gap) Step-by-step Every tool call Strategy docs, product planning
D: Prohibited Restricted None N/A PII, financial data, acquisition materials

Quality Gates

Stage Definition Gate Criteria Reviewer
Draft AI-assisted first pass Exists, roughly structured Self
Shaped Human-edited, logic verified Factually accurate, coherent Self + spot check
Review-Ready Ready for stakeholder input Complete, addresses known concerns Peer or stakeholder
Shippable Ready for external consumption Quality checklist passed, approved Designated approver

8. Risks & Mitigations

Risk Likelihood Impact Mitigation
Security/Legal blocks agentic policy extension Medium High Start with Sandbox tier; build trust incrementally; provide audit trail
Low adoption due to friction/complexity Medium Medium Start with high-value, low-friction use cases; measure and share wins
Quality issues undermine trust Medium High Enforce quality gates; train on output validation; visible review process
No clear ownership / CoE doesn't form Low High Secure executive sponsorship early; make ownership explicit in RACI
Policy gaps cause incident Low Very High Conservative default (Prohibited); explicit approval required for elevation
Maturity gap creates two-tier productivity High Medium Pair advanced users with teams; focus on raising the floor first
Success Metrics

Adoption: % of teams using approved AI workflows
Productivity: Time savings on common tasks (self-reported, then measured)
Compliance: Zero policy violations; 100% of outputs classified
Maturity: Average team maturity level (target: 3.5 by end of Q2)
Quality: % of AI-assisted outputs reaching "Shippable" without major revision
