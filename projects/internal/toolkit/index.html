Agentic Commerce Toolkit

Reference Patterns for Enterprise & Solution Architects

What This Is

A collection of reference patterns to support initial discovery conversations with clients exploring agentic commerce. Use these materials to speak intelligently about the problem space, identify key decisions, and scope engagement complexity — before knowing client-specific architecture.

Not for direct client delivery. These are conversation starters and scoping tools. All patterns require client-specific adaptation before becoming deliverables.

Discovery Questions

Questions to Ask in First Conversations

Current State

  • What commerce platform are you on today? (Salesforce, Adobe, SAP, custom)
  • Is your commerce architecture headless or monolithic?
  • Do you have existing APIs that external systems consume?
  • What's your current chatbot/virtual assistant situation?

Strategic Intent

  • What's driving interest in agentic commerce now?
  • Are you seeing traffic from AI search engines (Perplexity, ChatGPT)?
  • Is this defensive (don't lose share) or offensive (new channel)?
  • Who owns this initiative? (Digital, IT, Innovation, Strategy)

Technical Readiness

  • Do you have Schema.org structured data on your PDPs?
  • What's your latency tolerance for commerce APIs? (sub-second?)
  • How do you handle customer consent today?
  • What's your cloud platform? (AWS, GCP, Azure)

Constraints

  • Are there specific LLM platforms you need to support? (GPT, Claude, Gemini)
  • What are the compliance requirements? (PCI, GDPR, CCPA)
  • Is there a timeline driver? (competitor move, board mandate)
  • What's the budget envelope for this initiative?

Migration Path Selection Guide

Which Path Fits the Client?

Path A: Strangler Fig

  • Legacy monolithic commerce (SAP Hybris, Oracle ATG, custom)
  • Limited or no APIs; data locked in legacy DB
  • Can't replatform; need incremental migration
  • Timeline: 12-18 months

Path B: Additive

  • Already headless (Commercetools, BigCommerce, Shopify Plus)
  • APIs exist; just need agent layer on top
  • MACH architecture in place
  • Timeline: 3-6 months

Path C: API-First

  • Hybrid situation (monolith + some modern services)
  • Need to extract APIs before adding agents
  • API gateway exists or planned
  • Timeline: 6-12 months

Path D: Greenfield

  • No commerce platform today (new business line)
  • Building from scratch with modern stack
  • Can make agent-native architecture choices
  • Timeline: 6-9 months
