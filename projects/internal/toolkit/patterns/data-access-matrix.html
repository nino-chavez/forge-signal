Conditional Access Rules

Masked Data

  • Email: j***@example.com
  • Phone: ***-***-1234
  • Card: ****-****-****-1234

w/ Mandate

Payment mandate requirements:

  1. Customer has pre-authorized agent for payments
  2. Transaction amount within mandate limit (e.g., $500/transaction, $2000/month)
  3. Merchant category matches mandate scope
  4. Valid SPT (Secure Payment Token) exists
  5. Mandate not expired or revoked

w/ Approval

Requires additional confirmation:

  1. Agent must request explicit customer approval
  2. Approval via trusted channel (SMS, email, app push)
  3. Time-limited approval window (e.g., 5 minutes)
  4. Audit log entry created

Age-gated only

  • Only returns boolean (over 18, over 21)
  • Never returns actual date of birth

Implementation Checklist

  • Agent identity verification at API Gateway
  • Data classification tags on all API responses
  • Consent check before accessing consented/PII data
  • Mandate validation before payment operations
  • Audit log for all data access events
  • Rate limiting by agent type
  • Anomaly detection for access patterns
  • Quarterly access matrix review
  • Annual compliance audit
