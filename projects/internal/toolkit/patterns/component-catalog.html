Component Catalog

Reference specifications for scoping and complexity estimation

Reference Pattern

Catalog Structure

Each component includes:

  • Description: What it does
  • Responsibilities: Key functions
  • Interfaces: APIs and integrations
  • Vendor Options: Market alternatives (AI vendors named, others generic)
  • Complexity: Integration effort (S/M/L/XL)
  • Maturity: Production / Beta / Emerging
  • Phase: Day 1 / Phase 2 / Phase 3

1. External Agent Touchpoints

1.1 LLM Platforms

DescriptionThird-party LLM platforms that initiate commerce interactions
ResponsibilitiesNatural language understanding, intent extraction, response generation
InterfacesACP (OpenAI), MCP (Anthropic), AP2 (Google)
Vendor OptionsGPT-4/ChatGPT, Claude, Gemini, Llama (Meta)
ComplexityM (per protocol)
MaturityProduction
PhaseDay 1
NotesEach platform has different integration patterns; prioritize based on customer usage

1.2 AI Search Engines

DescriptionAI-powered search engines that surface commerce content
ResponsibilitiesContent discovery, citation, product recommendations
InterfacesStructured data (Schema.org), GEO-optimized content
Vendor OptionsPerplexity, Google AI Overviews, Bing Copilot
ComplexityS (content optimization)
MaturityBeta
PhasePhase 2
NotesPassive channel; focus on content optimization rather than API integration

1.3 Personal Agents

DescriptionConsumer-owned AI assistants with delegated authority
ResponsibilitiesAct on customer behalf across multiple services
InterfacesTo be defined (emerging standards)
Vendor OptionsEmerging (Apple Intelligence, Google Assistant, custom)
ComplexityXL (when available)
MaturityEmerging (2026+)
PhasePhase 3
NotesMonitor standards development; prepare agent registry for future integration
