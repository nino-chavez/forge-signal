02 Role Taxonomy Role Deep Dive

What each role does — and what differentiates strong candidates

Role Key Responsibilities Must-Have Skills Red Flags
Agentic Architect Design multi-agent systems; define agent boundaries; establish guardrails and safety; own technical decisions Distributed systems (10+ yrs); LLM internals; multi-agent patterns; production AI experience Only knows one framework; no production experience; can't discuss failure modes
Agent Engineer Implement agent logic; build tool libraries; integrate with enterprise systems; debug reasoning chains Python (strong); agent frameworks; async programming; API design; tool/function calling Can't explain tool design; no async experience; treats agents as chatbots
Prompt Engineer Craft system prompts; design prompt templates; A/B test variations; optimize for safety Writing clarity; structured thinking; LLM behavior understanding; evaluation methodology Thinks it's "just talking to AI"; no evaluation rigor; can't iterate systematically
Knowledge Engineer Design RAG pipelines; implement retrieval strategies; manage embeddings; curate knowledge quality Vector DBs (Pinecone, Weaviate); embedding models; chunking strategies; information retrieval Only knows one vector DB; ignores retrieval quality; no chunking strategy
Agentic QA Design behavioral test cases; build evaluation pipelines; red team agents; monitor production AI testing methods; evaluation metrics; adversarial testing; statistical analysis Expects deterministic outputs; only tests happy paths; no behavioral evaluation

8 / 16 Section 03: Build Strategy
03 Build Strategy Reskilling Programs

Structured training tracks with defined outcomes

FAST TRACK · 40 HRS

Prompt Engineering

For: Content strategists, UX writers, QA engineers

Week 1: LLM Fundamentals (8 hrs)

How LLMs work, tokenization, capabilities/limitations

Week 2: Prompt Patterns (16 hrs)

Zero/few-shot, CoT, system prompts, structured outputs

Week 3: Evaluation (16 hrs)

Criteria design, A/B testing, red teaming, capstone

Outcome: Independent prompt design capability

STANDARD TRACK · 120 HRS

Agent Engineering

For: Backend devs, full-stack engineers

Month 1: Foundations (40 hrs)

LLM APIs, agent frameworks, tool calling, memory

Month 2: Advanced (40 hrs)

Multi-agent orchestration, RAG, streaming, fallbacks

Month 3: Production (40 hrs)

Observability, testing, deployment, security, capstone

Outcome: Production agent system delivery

DEEP TRACK · 200 HRS

Agentic Architecture

For: Senior architects, tech leads

Month 1: Foundations (40 hrs)

Complete agent engineering track (accelerated)

Month 2-3: System Design (120 hrs)

Multi-agent architecture, enterprise patterns, safety

Month 4: Leadership (40 hrs)

Technical leadership, stakeholder comms, capstone

Outcome: Enterprise system design capability

Proficiency Definitions

Aware: Understands concepts, can discuss intelligently
Learning: Can complete guided tasks, needs supervision
Proficient: Works independently, delivers quality
Expert: Solves novel problems, mentors others

11 / 16 Section 04: Assessment
04 Assessment Interview Questions

Role-specific questions that differentiate strong candidates

Agent Engineer

"Design an agent that helps someone pick a TV. Walk me through the tool design."
"What's the difference between ReAct and function calling? When use each?"
"How do you handle an agent stuck in a loop?"
"Explain how you'd implement memory in a conversational agent."

Agentic Architect

"Design an agentic system for product discovery, comparison, and purchase."
"How would you architect multi-agent collaboration?"
"What are the key failure modes? How do you mitigate them?"
"How do you balance latency, cost, and quality?"

Prompt Engineer

"This agent keeps recommending wrong products. Here's the prompt. Fix it."
"Design a prompt for a multi-step task with constraints."
"How do you measure prompt quality? What metrics?"
"Walk me through your iteration process when a prompt fails."

Agentic QA

"How do you test something that gives different answers each time?"
"Design a test suite for a shopping assistant agent."
"What adversarial test cases would you create?"
"How do you establish success thresholds for agent behavior?"

Appendix Agentic vs. Traditional Development

Why existing teams can't just "figure it out"

Traditional Development Agentic Development
Deterministic outputs Probabilistic reasoning
Unit tests pass/fail Behavioral evaluation
Debug with logs Debug with traces & reasoning chains
Ship and monitor Ship, monitor, and iterate prompts
Edge cases are bugs Edge cases are training data
Users follow flows Users have conversations
Model training & fine-tuning Agent orchestration & tool design
Single-model optimization Multi-agent coordination

The Gap: Agentic AI requires a blend of software engineering, systems thinking, and domain expertise that doesn't exist in current ML or traditional software teams. This capability must be intentionally built.
